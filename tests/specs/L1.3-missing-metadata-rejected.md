# L1.3 — missing required metadata is rejected

**Test:** `test_L1_missing_metadata_rejected`
**Layer:** SHACL well-formedness · **Status:** GREEN (built)

## What it checks
A composition statement that LACKS the required metadata is REJECTED. The builder
re-emits a valid scenario with `full_metadata=False`, which strips: validity
period, provenance (source + agent), production technology — and SHACL must fail.

## Checked against
The `minCount 1` requirements on the shapes for `hasValidityPeriod` (or
`validFrom`), `prov:wasDerivedFrom`, `prov:wasAttributedTo`, `hasProduction`.

## Expected outcome
`conforms == False`, with at least one message containing "must have" or
"must record".

## Why it matters
These pieces of metadata were silently optional earlier (the shape only had
`maxCount`). This test guarantees they remain mandatory — the lesson from "the
shapes said conform while data was incomplete".

## Open questions
None. (If you later want a *more granular* test — one per missing property — say
so and it splits per required property.)
