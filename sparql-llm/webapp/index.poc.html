FutuRaM Chat
PoC · live

FutuRaM Chat answers questions about the material, component and element composition of products in the FutuRaM knowledge graph. It turns your question into a SPARQL query over the flat fq: query vocabulary and runs it against the live-compute proof-of-concept endpoint: the fq: graph is produced on the fly by the Python oracle rather than precomputed and stored in Jena.

Confirm the backend yourself by asking for the provenance node: SELECT ?p ?o WHERE { fq:_provenance ?p ?o } — only the live PoC endpoint carries it.

Ask, for example, how much of an element / material / component is in a given class — the assistant builds the fq:contains pattern, declares the constituent kind via the ontology (rdfs:subClassOf futuram:Element), and reads back the amount.
